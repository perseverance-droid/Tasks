## ***GitHub 的分享***

### 1.GitHub 是做什么的

GitHub 是一个面向开源及私有软件项目的托管平台。简单来说，它就像是一个巨大的“代码仓库”，开发者可以把自己的项目代码放在上面，进行版本管理、协作开发等操作。

> **Fork：可以理解为“复制”别人的仓库到自己的 GitHub 账户下。这样，你就可以在不影响原仓库的情况下，对这个项目进行自己的修改和开发。比如，你看到一个有趣的开源项目，想在此基础上做一些个性化的调整，就可以先 Fork 这个项目的仓库。**

> **Pull request：当你在自己 Fork 后的仓库里对项目进行了修改，想要把这些修改合并到原项目的仓库中时，就可以发起一个 Pull request。原项目的维护者会查看你的修改内容，如果觉得合适，就会把你的修改合并到原仓库，这样你的贡献就被纳入到原项目中了。**

### 4.文档整洁规范的理由与遵循的规范

(一)理由

通过合理运用 Markdown 语法，设置了不同等级的标题,并且使不同的部分有明确的标题，内容条理清晰，读者能够很容易地找到自己所需的信息，所以称得上整洁规范。

(二)遵循的规范

1. 标题层级规范：使用  #  来表示不同层级的标题，一级标题用  # ，二级标题用  ## ，这样可以清晰地体现内容的层级结构。
2. 列表规范：在介绍 Pull request、Fork 等内容以及学习途径时，使用有序列表（ 1.   2.   3. ），让内容更有条理。
3. 强调规范：对于“Geek 精神”等重点内容，使用加粗（ **** ）的方式进行强调，突出关键信息。
